## User Stories for Sprint 1
The following user stories have been identified for Sprint 1:

### 1. *User Story 1: User Registration & Login*
- *As Ayaan*, I want to quickly create an account and log in, so that I can save my progress and return to my simulated portfolio anytime.
- *Acceptance Criteria*:
  - A registration form with email, password, and username fields.
  - Validation for empty or invalid input.
  - Error message when login credentials are incorrect.
  - Redirect to the dashboard after successful login.
  - Session persists until logout (user stays logged in).

### 2. *User Story 2: Simulated Stock Trading (Buy/Sell)*
- *As Mina*, I want to buy and sell virtual stocks using fake money, so that I can practice trading without real financial risk.
- *Acceptance Criteria*:
  - Display a list of available stocks with prices that update periodically.
  - User can “buy” or “sell” a stock using their virtual balance.
  - Show transaction confirmation and update portfolio immediately.
  - Prevent invalid trades (e.g., not enough funds or negative quantities).
  - Display feedback messages for successful or failed trades.

### 3. *User Story 3: Portfolio Overview Dashboard*
- *As Ethan*, I want to see a clear summary of my holdings and performance, so that I can track how my simulated investments are doing.
- *Acceptance Criteria*:
  - Show total balance, profit/loss, and list of owned stocks.
  - Each stock shows name, quantity, buy price, and current value.
  - Include a small performance chart (e.g. daily gain/loss).
  - Responsive layout for desktop and mobile.
  - Navigation links to trade, tutorials, and settings.

### 4. *User Story 4: Learning Resources Hub*
- *As Jess*, I want access to short, beginner-friendly guides and videos about trading basics, so that I can learn while I practice.
- *Acceptance Criteria*:
  - Page with categorized topics (e.g. “How Stocks Work”, “Risk Management”, “Understanding Charts”).
  - Each topic opens a short article or embedded video.
  - Mark lessons as “completed” to track learning progress.
  - Integrate a link from the dashboard sidebar to “Learn”.
  - Responsive design for desktop and mobile.
